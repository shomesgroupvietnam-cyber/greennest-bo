# Implementation Artifacts

Use this folder for BMad implementation evidence after a request is ready to build.

Expected artifacts include:

- approved implementation specs
- story execution notes
- sprint status snapshots
- checkpoints
- investigations
- code review reports
- QA automation notes
- retrospectives

Implementation artifacts should record what was built, why it was built and how it was verified. Long-term decisions should still be promoted to `blueprint/` or `docs/`.
